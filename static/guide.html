Your 30-second pitch

"NEXUS is a governed memory and knowledge layer that sits between your AI agents and your data. Instead of every team building their own RAG pipeline, NEXUS gives you one central place — with intelligent routing, a knowledge graph, tiered memory, and access control baked in. I built the full platform from scratch in Python: from the vector store to the query pipeline to the governance layer."

The Anti-RAG Router — routing strategies

StrategyWhat it meansWhat happens
GRAPH Query mentions a known entity (person, team…) Knowledge graph traversal first, then retrieve chunks linked to those entities
HYBRID Mixed / broad question All three retrievers fire: dense + BM25 + graph → merged with RRF
DIRECT Factual / math question Knowledge store skipped entirely — answer returned directly
STRUCTURED Structured data query Skip unstructured retrieval, route to structured lookup
CACHED Query seen recently Return cached result, zero latency

Chunk cards — what the badges mean

Domain badge

Which data domain the chunk belongs to — SPORTS (blue), COMPLIANCE (yellow), OPERATIONS (red), PRODUCT (violet). Tells you what kind of knowledge was retrieved.

Sensitivity badge

PUBLIC (green) → INTERNAL (blue) → CONFIDENTIAL (yellow) → RESTRICTED (red). This is the classification that ABAC checks against the agent's clearance level.

RRF score

Reciprocal Rank Fusion score — how relevant the chunk is. Computed by merging the rank positions from dense, sparse, and graph retrievers, not their raw similarity scores.

Source tags

dense / sparse / graph — which of the three retrievers found this chunk. A chunk with all three tags was ranked high by every method.

Technical talking points

If the interviewer asks "how does this work under the hood":

No double-retrieval

The retriever stores last_strategy, last_results, and last_excluded as instance variables. The API reads them after agent.think() returns — no second retrieval, no double audit entries.

RRF Fusion

Dense (cosine similarity), BM25 (term-frequency sparse), and Graph RAG (BFS entity expansion) each produce ranked lists. RRF merges them by rank position — robust to score scale differences between retrievers.

KG relationships stored twice

KnowledgeGraph.adj stores (target_id, weight) for BFS. A separate _rels list stores full Relationship objects with relation_type — the UI reads those to draw labeled edges on the canvas.

Python 3.8 compatible

All files use from __future__ import annotations and the typing module throughout — no 3.10+ union syntax or 3.9+ generic collections. Runs on stock Anaconda Python.

Zero external dependencies

HashingEmbedder (stdlib only) and InMemoryVectorStore mean the entire platform runs without pip installs. sentence-transformers and anthropic are optional upgrades.

Tiered memory

Working memory (10-slot ring buffer) → EpisodicMemory (compresses old traces via extractive summarization) → SemanticMemory (vector-indexed for recall). All three tiers are coordinated by MemoryManager.

If something breaks during recording

Server not running / connection refused

cd d:\deeplearning\Nexus\nexus_platform
python -m uvicorn app:app --port 8000

Port 8000 already in use

python -m uvicorn app:app --port 8001

Then go to http://localhost:8001

Browser showing old cached version

Hard refresh: Ctrl + Shift + R

Chunks not appearing after restart

The server uses in-memory storage — a restart resets it. Just restart uvicorn and everything re-seeds automatically on startup.
